# C8 · Crunch Labs — Web Dev · Syllabus

**12 weeks · ~36 hrs/week intensive (or scaled) · Beginner → frontend-job ready**

A free, open-source 12-week frontend track. HTML through accessible, animated, deployed apps. No frameworks required to start; we add a thoughtful sprinkle of modern tooling once the fundamentals are solid.

---

## Program at a glance

| Phase | Weeks | Outcome |
|-------|-------|---------|
| **Phase 1 — Markup & Style** | 01 – 03 | Semantic HTML, modern CSS, responsive layout, accessibility |
| **Phase 2 — Interaction & Logic** | 04 – 06 | JavaScript, the DOM, events, forms, validation |
| **Phase 3 — Modern Tooling** | 07 – 09 | Build tools, components, state, fetch, animation |
| **Phase 4 — Production** | 10 – 12 | Deployment, performance, accessibility audits, capstone |

---

## Weekly breakdown

### Phase 1 — Markup & Style

**Week 0 — Setup.** Editor, terminal, browser dev tools, GitHub. A clean working environment.

**Week 1 — Semantic HTML.** Document outline, headings, landmarks, lists, links, images with alt, forms, ARIA basics.

- *Mini-project:* A 5-page personal site, semantic HTML only, no CSS yet.

**Week 2 — Modern CSS.** Custom properties, the cascade, specificity, box model, typography that doesn't fight the system.

- *Mini-project:* Style the Week-1 site. No CSS frameworks. Honest CSS only.

**Week 3 — Layout & responsiveness.** Flexbox, Grid, container queries, breakpoints, the mobile-first mindset.

- *Mini-project:* The Week-2 site reflows cleanly from 320 px to 1440 px.

### Phase 2 — Interaction & Logic

**Week 4 — JavaScript fundamentals.** Types, control flow, functions, scope, closures, modules. No frameworks.

- *Mini-project:* A vanilla-JS to-do list with localStorage persistence.

**Week 5 — The DOM & events.** Querying, mutating, delegating, accessibility, focus management.

- *Mini-project:* An accessible custom dropdown, keyboard-navigable.

**Week 6 — Forms & validation.** Form semantics, HTML5 validation, JS validation, error messaging, autofill.

- *Mini-project:* A multi-step signup form with real-time validation.

### Phase 3 — Modern Tooling

**Week 7 — Build tools & components.** Vite, ES modules, component patterns (web components or a light framework — we recommend Astro for content, or React for interactive). The minimum tooling that helps.

- *Mini-project:* Move the Week-6 form into a component-based codebase.

**Week 8 — Fetch, state, and async.** `fetch`, JSON, loading/error states, caching, optimistic updates.

- *Mini-project:* A weather widget consuming a free public API.

**Week 9 — CSS animation & motion.** Transitions, keyframes, the Web Animations API. `prefers-reduced-motion`. The ethics of motion.

- *Mini-project:* A landing page with three motion touches, all respecting reduced-motion preferences.

### Phase 4 — Production

**Week 10 — Deployment & performance.** Vercel / Netlify deploy, Lighthouse, Core Web Vitals (LCP, INP, CLS), image optimization, font loading.

- *Mini-project:* Lighthouse score ≥ 95 on the Week-9 page.

**Week 11 — Accessibility audit.** WCAG 2.2 AA practical checklist. Screen readers (VoiceOver / NVDA). Keyboard-only navigation. Color contrast.

- *Mini-project:* Audit and fix three issues in the Week-9 page. Ship the audit report.

**Week 12 — Capstone.** A 4-page interactive site — your portfolio, a small product page, a redesign of a public-domain content piece, your choice — applying every prior week.

- *Capstone:* Public deploy + Lighthouse ≥ 95 + accessibility audit + post-mortem.

## What this track is NOT

- **Not a fullstack course.** Backend is C16 (Python) or C9 (.NET).
- **Not a React-only course.** We teach the platform first. Frameworks are tools.
- **Not a design course.** We teach implementation. Design fundamentals are an adjacent specialization.
